Binary conversion




Remember how we ended the Decimal-to-Binary conversion?


That's right: by inverting the order of all those 0s and 1s.


If the true scientific reason behind this inversion is of no real concern to us, what we must remember is the notion of INDICES.


Let's look at the following binary expression:


1 0 1 0 0 0 0 1 1


An INDEX is allocated to each of these numbers, with the rightmost number bearing the index 0.


The binary expression with all its INDICES will now look like this:


18 07 16 05 04 03 02 11 10


So what on earth are these INDICES for???


Simple: each of these INDICES is used as an exponent of 2 (since binary expressions consist of BASE 2 values).


Let's illustrate this with an example.


We start with the leftmost digit:


18 07 16 05 04 03 02 11 10


...before following these simple steps:


"Is the binary digit a 1?"

18 07 16 05 04 03 02 11 10

Yes!

Therefore we do the following operation:

28 = 256


"Is the next binary digit a 1?"

18 07 16 05 04 03 02 11 10

No!

We move on.


"Is the next binary digit a 1?"

18 07 16 05 04 03 02 11 10

Yes!

Therefore we do the following operation:

26 = 64


"Is the next binary digit a 1?"

18 07 16 05 04 03 02 11 10

No!

We move on.


"Is the next binary digit a 1?"

18 07 16 05 04 03 02 11 10

No!

We move on.


"Is the next binary digit a 1?"

18 07 16 05 04 03 02 11 10

No!

We move on (I think you're getting the idea...)


"Is the next binary digit a 1?"

18 07 16 05 04 03 02 11 10

No!

We move on (yup, I think you do...)


"Is the next binary digit a 1?"

18 07 16 05 04 03 02 11 10

Yes!

Therefore we do the following operation:

21 = 2


"Finally: is the next binary digit a 1?"

18 07 16 05 04 03 02 11 10

Yes!

Therefore we do the following operation:

20 = 1


We now come to the exciting part: we add all of the values found above:


256 + 64 + 2 + 1 = 323


Recap time!


Converting a binary expression to a decimal value consists of the following three steps:


  1. assign an index to every digit of the binary expression (starting with index 0 on the rightmost digit)
  2. use that index number as an exponent of 2 everytime the binary digit is a 1
  3. finally, add all the values found, which gives you the decimal value of the binary expression!

In this chapter, we only looked at how to convert a binary expression to a decimal.


To convert to the other two sorts (octal & hexadecimal), simply use the decimal result of your binary-to-decimal conversion and follow the techniques illustrated in the Decimal chapter:
