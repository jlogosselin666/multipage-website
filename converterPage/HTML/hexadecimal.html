Hexadecimal conversion




Before we start examining hexadecimal expressions, it is important to do a brief recap of what we have encountered so far.


We have seen that decimal expressions have 10 numbers ( 0-9 - hence: BASE 10); binary expressions have 2 numbers (1 + 0 - hence: BASE 2); octal expressions have 8 numbers (0-7 - hence: BASE 8).


However: hexadecimal expressions have 16 characters altogether ( 0-9 and A-F - hence: it is base BASE 16).


Therefore, to convert hexadecimal expressions to decimal, the same technique seen before for binary expressions and octal expressions is used: the index becomes the exponent of 16, and the result of this operation is then multiplied with the number to which that index belongs.


Let's have a look at an example.


Say we have the following hexadecimal expression:


A 9 F 3 1 D


With INDICES, the hexadecimal expression now looks like this:


A5 94 F3 32 11 D0


As usual, we start with the leftmost digit:


A5 94 F3 32 11 D0


First, we set the index 5 as exponent to base 16:

165 = 1,048,576

We then multiply this value by the number to which the "index/exponent" is attached...

...but wait! Didn't we see in the hexadecimal table (see lesson on decimals) that character A is equivalent to 10?

Therefore, instead of doing: 1,048,576 x A, we do:

1,048,576 x 10 = 10,485,760


We move on to the next digit:


A5 94 F3 32 11 D0


We set the index 4 as exponent to base 16:

164 = 65,536

We then multiply this value by the number to which the "index/exponent" is attached:

65,536 x 9 = 589,824


We move on to the next digit:


A5 94 F3 32 11 D0


We set the index 3 as exponent to base 16:

163 = 4,096

We then multiply this value by the number to which the "index/exponent" is attached...

...but wait! Didn't we see in the hexadecimal table (see lesson on decimals) that character F is equivalent to 15?

Therefore, instead of doing: 4,096 x F, we do:

4,096 x 15 = 61,440


We move on to the next digit:


A5 94 F3 32 11 D0


The index 2 is set as exponent to base 16:

162 = 256

We then multiply this value by the number to which the "index/exponent" is attached:

256 x 3 = 768


We move on to the next digit:


A5 94 F3 32 11 D0


The index 1 is set as exponent to base 16:

161 = 16

We then multiply this value by the number to which the "index/exponent" is attached:

16 x 1 = 16


We move on to the last and final digit:


A5 94 F3 32 11 D0


We set the index 0 as exponent to base 16:

160 = 1

We then multiply this value by the number to which the "index/exponent" is attached...

...but wait! Didn't we see in the hexadecimal table (see lesson on decimals) that character D is equivalent to 13?

Therefore, instead of doing: 1 x D, we do:

1 x 13 = 13


We now add all of the values found above:


10,485,760 + 589,824 + 61,440 + 768 + 16 + 13 = 11,137,821


Recap time!


Converting a hexadecimal expression to a decimal value consists of the following steps:


  1. assign an index to every digit of the hexadecimal expression (starting with index 0 on the rightmost digit)
  2. use that index number as an exponent, this time of base 16 (bearing in mind that hexadecimal values range from 0-9 and from A-F)
  3. multiply the result with the number to which is attached the "index/exponent"
  4. finally, add all the values found, which gives you the decimal value of the hexadecimal expression!

In this chapter, we only looked at how to convert a hexadecimal expression to a decimal.


To convert to the other two sorts (binary & octal), simply use the decimal result of your hexadecimal-to-decimal conversion and follow the techniques illustrated in the Decimal chapter:
